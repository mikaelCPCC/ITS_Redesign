How to use the Online Service Avalibility System

All of these actions must be performed using the HTML editor and not the rich text editor. We recommend using a third-party program such as Notepad++ or Sublime Text to make these changes as it will be much easier to follow the code outside of the Plone editing enviorment. Simply copy and paste the source HTML in and out of Plone and into the third party editor.

Changing the icon status indicator.

  • To change the icon between different service states, locate the service you will need to change. This is indicated by an HTML comment at the top of the section (e.g., <-- BLACKBOARD SECTION -->).
  • Locate the <img> tag (e.g., <img src="http://webstaging.cpcc.edu/its/images/green/">).
  • Simply change the color name after the /images/osa/ markup to the icon you want to display on the page.

  • Changing the color will only change the icon. Please see below for instructions on changing the message.

Changing the broadcast message

All messages are universal and controlled through the use of HTML comment tags. Comment tags control what can be seen by the public and what cannot. Here is an example of a comment tag:
<!--This is a comment. Comments are not displayed in the browser-->
You'll notice that every broadcast message is listed in each section under the SET STATUS HERE. REMOVE COMMENT TAGS AS NEEDED HTML comment. The four broadcast messages are as follows:

  • This service is currently available with no known issues. (for use with green icon)
  • This service is available but some users have reported a problem. Please contact the help desk if you are experiencing issues using this service. (for use with the yellow icon)
  • This service is currently not available and the problem is being addressed. Please check this page for updates. (for use with the red icon)

The default broadcast message is: This service is currently available with no known issues.
If you need to change to another message, simply remove the

<!--
from the beginning of the message you want to use, and
--> from the end.
The message is now active.

To deactivate a broadcast message simply add <!-- to the front of the message, and --> to the end of the message.


When the service returns to a normal state, simply reverse the above the instructions to set it back to the default notification.
